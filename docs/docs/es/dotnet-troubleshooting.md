## Solución de Problemas

### Problemas Comunes

1. **El recurso Dev Tunnel falló al iniciarse**: Asegúrate de haber ejecutado `devtunnel login` y autenticado tu cuenta con Microsoft Dev Tunnels.
1. **Hay recursos faltantes listados en el panel de Aspire**: Verifica que hayas completado los pasos de [Comenzando](./getting-started.md) y que todos los recursos estén desplegados en Azure. Verifica que los parámetros hayan sido agregados a los secretos de usuario abriendo una ventana de Terminal y ejecutando `dotnet user-secrets list` en el directorio del proyecto AppHost. Debe contener los siguientes valores:

   ```
    Parameters:ResourceGroupName = ...
    Parameters:pg-password = ...
    Parameters:mcp-inspector-proxyToken = ...
    Parameters:FoundryResourceName = ...
    Parameters:FoundryProjectName = ...
    Parameters:FoundryEndpoint = ...
    Parameters:EmbeddingModelDeploymentName = ...
    Parameters:ChatModelDeploymentName = ...
    Parameters:AzureOpenAIEndpoint = ...
    ConnectionStrings:ApplicationInsights = ...
    Azure:Tenant = ...
    Azure:SubscriptionId = ...
    Azure:ResourceGroup = ...
    Azure:Location = ...
   ```

1. **dotnet no soportado al lanzar el depurador**: Asegúrate de instalar la extensión de C# en Visual Studio Code. Si ya la has instalado, intenta reiniciar Visual Studio Code.
   ![Error del depurador](../media/troubleshooting/dotnet-debugger-not-supported.png)

1. **El panel de Aspire no carga**: Si el navegador no carga, intenta refrescar completamente la página (Ctrl + F5 o Cmd + Shift + R). Si aún no carga, asegúrate de que el Dev Tunnel esté ejecutándose verificando la ventana de terminal donde iniciaste el Dev Tunnel. Si no está ejecutándose, navega al directorio del proyecto AppHost y ejecuta `dotnet run` para iniciarlo nuevamente.

1. **El navegador no se lanzó con el depurador**: Verifica la Consola de Depuración de VS Code para la URL `http://localhost:15184`. El panel de Aspire usa una clave generada aleatoriamente para autenticación, la cual está incluida en la URL. Copia y pega la URL en tu navegador para acceder al panel de Aspire. El mensaje de terminal se ve así:

   ```
   info: Aspire.Hosting.DistributedApplication[0]
         Now listening on: http://localhost:15184
   Aspire.Hosting.DistributedApplication: Information: Now listening on: http://localhost:15184
   info: Aspire.Hosting.DistributedApplication[0]
         Login to the dashboard at http://localhost:15184/login?t=<token>
   ```

1. **El recurso `ai-foundry` reporta _Failed to provision_.**: Esto puede suceder si la conexión de Azure falla al establecerse en la primera ejecución de la aplicación. Reinicia el depurador en Visual Studio Code para intentar nuevamente.
